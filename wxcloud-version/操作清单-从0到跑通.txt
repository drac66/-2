微信小程序（腾讯云/微信云开发）完整操作清单

【目标】
把当前项目 wxcloud-version 跑通：登录、留言、日记、相册上传。

====================
一、准备阶段（先做）
====================
1）确保微信开发者工具能打开
2）确认当前环境 ID：dsx57-d1gpxn8u87a6ddbb8
3）确认项目目录：
   - 小程序前端：wxcloud-version/miniprogram
   - 云函数目录：wxcloud-version/cloudfunctions


====================
二、导入项目
====================
1）打开微信开发者工具
2）导入项目，选择目录：wxcloud-version/miniprogram
3）确认 appid 正确（你当前项目 appid）
4）进入项目后，确认能看到页面（login/home/album/diary/messages）


====================
三、部署云函数（必须）
====================
依次部署以下 5 个函数（上传并部署：云端安装依赖）：
1）cloudfunctions/wxlogin
2）cloudfunctions/messages
3）cloudfunctions/diary
4）cloudfunctions/albumsUpload
5）cloudfunctions/albumsList

每部署完一个，观察是否提示成功。

成功标准：
- 控制台里能看到 5 个函数
- 没有 Function not found 报错


====================
四、创建数据库集合（必须）
====================
在云开发控制台 -> 文档型数据库，新建以下集合：
1）tokens
2）messages
3）diary
4）albums

说明：
- 先用开发期默认权限跑通
- 后续上线前再收紧权限


====================
五、逐项功能测试（按顺序）
====================
A. 登录测试
1）打开登录页
2）点击登录
3）预期：跳转到 home
4）检查 tokens 集合是否新增记录（含 token / openid）

B. 留言测试
1）进入 messages 页
2）输入一条消息并发送
3）预期：页面列表显示新消息
4）检查 messages 集合有新文档

C. 日记测试
1）进入 diary 页
2）输入标题和内容，选择可见性（self 或 both）
3）点击保存
4）预期：列表出现新日记
5）检查 diary 集合有新文档

D. 相册测试
1）进入 album 页
2）选择图片并上传
3）预期：显示上传成功，列表可见图片
4）检查 albums 集合有新文档
5）检查云存储存在 albums/ 开头文件


====================
六、常见报错排查
====================
1）Error: timeout
- 云函数没部署
- envId 不对
- 网络波动

2）Function not found
- 对应云函数未部署成功

3）unauthorized
- token 无效/不存在，重新登录

4）上传成功但不显示
- getTempFileURL 失败
- 数据库记录 media_url 不是 cloud://fileID


====================
七、成本控制（先按免费额度跑）
====================
1）只用当前环境，不额外新建环境
2）图片先限制大小（建议单图 <= 10MB）
3）减少频繁刷新，避免无意义调用
4）后续再加图片压缩


====================
八、最后收口（上线前必须）
====================
1）拿到你和女朋友两个 openid
2）修改 cloudfunctions/config.js：
   - openidWhitelist 改成真实 openid
   - enforceWhitelist 改为 true
3）重新部署 wxlogin 云函数
4）数据库权限改最小权限


====================
九、每次开发前自检（30秒）
====================
[ ] app.js 的 envId 正确
[ ] 5 个云函数已部署
[ ] 4 个集合存在
[ ] 登录可成功
[ ] 关键页面无红色报错
